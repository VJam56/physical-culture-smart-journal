# Implementation Specifications

**Project Version:** 1.3.0  
**Status:** Active

The `specs/` directory contains implementation contracts.

Documentation in `docs/` explains product intent, principles, architecture, and rationale.

Specifications in `specs/` define the behavior an engineer or implementation agent must produce.

## Specification Rules

Each specification must include:

- Purpose
- In scope
- Out of scope
- Inputs
- Outputs
- Data contracts
- Required behavior
- Validation
- Failure behavior
- Security and privacy
- Acceptance criteria
- Test cases
- Open dependencies

## Numbering

Specifications use stable numeric identifiers.

- `001_excel_import.md`
- `002_database_schema.md`
- `003_training_session.md`
- `004_guided_journal.md`
- `005_ai_observation.md`

Numbers identify the specification and should not change when titles are refined.
