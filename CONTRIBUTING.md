# Contributing

## Development Philosophy

1. Preserve MVP scope.
2. Document major decisions.
3. Prefer simple, testable implementations.
4. Separate observation from diagnosis.
5. Maintain user ownership and privacy.
6. Update documentation whenever behavior changes.

## Branch Naming

- `feature/<name>`
- `fix/<name>`
- `docs/<name>`
- `refactor/<name>`

## Commit Style

Use concise, descriptive commits.

Examples:

- `docs: define guided journal model`
- `feat: add session set logging`
- `fix: preserve set order during import`
- `refactor: normalize exercise identifiers`
