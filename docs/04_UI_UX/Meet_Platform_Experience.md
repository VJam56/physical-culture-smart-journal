# Meet Platform Experience

**Project Version:** 1.3.2  
**Status:** Approved UX Direction

## Design Metaphor

The primary training screen should feel like walking onto a well-organized platform at a powerlifting meet.

At a well-run platform:

- The bar is loaded.
- The rack is set.
- The attempt is known.
- Commands are clear.
- The lifter's responsibility is execution.

The app should create the same operational clarity during training.

## Primary Screen Hierarchy

### First Priority

- Current exercise
- Current set
- Planned target
- Actual-entry controls
- Complete-set action

### Second Priority

- Microphone
- Camera
- Short text note
- Rest status
- Next-set preview

### Third Priority

- Full session history
- Analytics
- Trends
- Long-form notes
- Settings

Third-priority information must not compete visually with the current objective.

## Session Rhythm

The interface should support:

Prepare → Execute → Capture → Recover → Next objective

It should not introduce:

Prepare → Navigate → Search → Re-enter → Save → Organize → Return to training

## Cognitive Load Test

For every visible control, ask:

- Does it help execute the current objective?
- Does it reduce administrative burden?
- Does it preserve evidence that would otherwise disappear?

A control that fails all three tests should not appear in the primary training view.

## Acceptance Criteria

- The current objective is identifiable at a glance.
- A set can be recorded without leaving the primary session view.
- Evidence can be captured before the next set overwrites it.
- Analysis does not interrupt the training rhythm.
- Repeated information is preloaded.
- The interface becomes quieter, not busier, as training intensity rises.
