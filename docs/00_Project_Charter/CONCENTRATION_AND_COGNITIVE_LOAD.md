# Concentration and Cognitive Load Principle

**Project Version:** 1.3.2  
**Status:** Approved

## Core Principle

The application preserves the athlete's cognitive and psychological resources for performance.

The training environment should feel like a well-organized competition platform:

- The objective is visible.
- Required information is ready.
- Record keeping recedes into the background.
- The athlete does not organize the system while trying to perform.

> Every interaction should either reduce cognitive load or preserve coaching-quality evidence.

## Operational Standard

### Before the Set

The application prepares:

- Exercise
- Set number
- Planned load
- Repetitions
- RPE or RIR target
- Rest guidance
- Relevant notes

### Immediately After the Set

The application enables:

- One-action completion
- Fast actual-value entry
- Short voice capture
- Camera capture
- Minimal technical annotation

### Between Sets

The application should:

- Preserve the next objective
- Avoid unnecessary summaries
- Avoid distracting analysis
- Save automatically
- Keep the athlete inside the active session

### After the Session

The application may:

- Reconstruct the journal
- Organize media
- Compare planned and actual work
- Generate historical summaries
- Present observations and uncertainty

## Governing Phrase

> Capture first. Organize later.

The app is not asking the athlete to become a more diligent administrator. It removes administrative work so the athlete can give greater cognitive and psychological priority to the lift.
