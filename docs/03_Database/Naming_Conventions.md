# Naming Conventions

## Database

- `snake_case`
- Singular table names or consistent plural table names; choose once before implementation
- Primary keys use `<entity>_id`
- Timestamps use UTC
- Boolean names begin with `is_`, `has_`, or `can_`

## Code

- Classes: `PascalCase`
- Functions and variables: language convention
- Avoid unexplained abbreviations
- Preserve domain language used in product documentation
